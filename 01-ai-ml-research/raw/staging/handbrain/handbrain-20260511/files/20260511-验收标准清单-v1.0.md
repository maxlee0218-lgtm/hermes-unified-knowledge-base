> 版本：v1.0
> 日期：2026-05-11
> 制定者：贾维斯（管家）
> 状态：生效

---

## 一、验收层级

### 1.1 四级验收体系

```
Agent自检 → 贾维斯初验 → 验收官审计 → 用户终验
```

| 层级 | 检查者 | 检查内容 | 通过标准 |
|------|--------|---------|---------|
| **自检** | Agent自己 | 完整性、准确性 | 无遗漏、无错误 |
| **初验** | 贾维斯 | 目标达成、质量合格 | 目标100%达成 |
| **审计** | 验收官 | 标准符合性、规范性 | 符合规范 |
| **终验** | 用户 | 满意度、价值 | 用户满意 |

---

## 二、验收检查项

### 2.1 任务完成度

| 检查项 | 权重 | 标准 |
|--------|------|------|
| 目标达成 | 30% | 所有目标已完成 |
| 范围覆盖 | 20% | 所有要求已覆盖 |
| 质量合格 | 25% | 无明显错误 |
| 时间合规 | 15% | 在约定时间内完成 |
| 文档完整 | 10% | 所有文档已输出 |

### 2.2 输出质量

| 检查项 | 标准 |
|--------|------|
| 结论明确 | 有明确的结论或结果 |
| 证据充分 | 有数据、日志或截图支撑 |
| 逻辑清晰 | 推理过程合理、无漏洞 |
| 建议可行 | 建议具体、可执行 |
| 格式规范 | 符合模板和格式要求 |

### 2.3 技术规范

| 检查项 | 标准 |
|--------|------|
| 代码规范 | 符合编码规范（如有） |
| 安全合规 | 无安全隐患 |
| 性能达标 | 满足性能要求（如有） |
| 可维护性 | 易于理解和维护 |

## 三、验收流程

### 3.1 标准流程

```
1. Agent完成任务
   ↓
2. Agent自检（对照检查项）
   ↓
3. Agent输出报告
   ↓
4. Agent评论请求验收
   ↓
5. 贾维斯初验（24小时内）
   ├─ 不合格 → 返回Agent补充
   └─ 合格 → 提交验收官
      ↓
   6. 验收官审计（48小时内）
      ├─ ACCEPT → 进入终验
      ├─ REWORK → 返回Agent修改
      └─ BLOCKED → 标记原因
         ↓
      7. 用户终验
         ├─ 通过 → 归档done
         └─ 不通过 → 返回修改
```

### 3.2 快速流程（简单任务）

```
1. Agent完成任务
   ↓
2. 贾维斯初验
   ├─ 合格 → 直接归档
   └─ 不合格 → 返回补充
```

**适用条件**：
- 任务简单，无风险
- 输出明确，易验证
- 用户授权快速验收

## 四、验收Verdict

### 4.1 Verdict类型

| Verdict | 含义 | 动作 |
|---------|------|------|
| **ACCEPT** | 验收通过 | 归档done，沉淀知识 |
| **REWORK** | 需要修改 | 返回Agent，明确修改要求 |
| **BLOCKED** | 无法继续 | 标记原因，等待解决 |
| **PASS** | 快速通过 | 简单任务直接归档 |

### 4.2 Verdict输出格式

```
【验收报告】
任务：XXX
验收人：XXX
时间：XXX

 verdict：ACCEPT/REWORK/BLOCKED

检查项：
- [x] 目标达成
- [x] 范围覆盖
- [x] 质量合格
- [x] 时间合规
- [x] 文档完整

评价：
优点：XXX
不足：XXX

建议：XXX
```

## 五、验收标准（按任务类型）

|--------|------|
| 研究深度 | 覆盖主要观点和证据 |

| 引用规范 | 引用来源可靠 |

| 结果可信 | 结果可复现 |

| 检查项 | 标准 |
|--------|------|
| 操作规范 | 符合运维规范 |
| 安全合规 | 无安全隐患 |
| 记录完整 | 操作记录完整 |
| 可回滚 | 重要操作可回滚 |

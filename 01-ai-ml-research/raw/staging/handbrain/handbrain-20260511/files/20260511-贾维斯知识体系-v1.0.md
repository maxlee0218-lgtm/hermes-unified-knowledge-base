## 一、数仓架构体系

### 1.1 分层模型
- **Bronze（原始数据）**：MES等业务系统原始数据
- **Silver（清洗数据）**：经过清洗、转换的数据
- **Gold（汇总数据）**：按主题汇总的数据
- **ADS（应用数据）**：面向应用的数据接口层

### 1.2 技术栈
| 组件 | 用途 | 版本 | 位置 |

| dbt | 数据转换 | - | Windows测试环境 |
| DuckDB | 本地分析引擎 | - | Windows测试环境 |
| DataX | 数据同步 | - | V2生产/V3升级目标 |
| DolphinScheduler | 调度系统 | 3.1.5(V2)/3.4.1(V3) | V2生产/V3升级目标 |
| PolarDB | 生产数据库 | MySQL 8.0 | 阿里云 |

### 1.3 项目结构
```
D:/data-warehouse/ (Windows测试环境)
├── 01-dbt-duckdb/     # dbt项目 + DuckDB(15GB+)
```

## 五、安全规范

### 5.1 生产保护原则
- **只读查询**：禁止写库、删库、改表
- **禁止改调度**：生产DolphinScheduler/DataX不可修改
- **禁止改配置**：生产环境配置不可修改
- **密钥脱敏**：密码、token、私钥不可明文打印

### 5.2 权限边界
- hw_dev_02：仅有SELECT权限
- 无INDEX、ALTER、PROCESS、ANALYZE权限
- Windows环境：用户profile混用（需整理）

## 八、贾维斯能力边界

### 8.1 能做的
- 读取多卡评论和本地报告
- 整理分析历史数据
- 创建任务和指派Agent
- 生成报告和清单
- 设计方案和架构

### 8.2 不能做的
- 直接连接Windows电脑
- 直接连接生产数据库
- 保证Agent一定成功执行
- 实时获取最新数据
- 控制网络连接（guest Wi-Fi）

### 8.3 需要用户协助的
- 提供最新数据库状态
- 确认Agent执行结果
- 提供文件内容（当Agent失败时）
- 决策关键操作（写库、改配置等）
- 连接guest Wi-Fi后执行查询

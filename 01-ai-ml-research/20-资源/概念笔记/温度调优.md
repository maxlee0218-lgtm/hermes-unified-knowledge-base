# Temperature 调优

## 原理

Temperature 是对 softmax 输出 logits 的重新加权。温度趋近于 0 时输出趋近贪婪采样，趋近无穷大时趋近均匀分布。

## 实战温度策略

| 场景 | 推荐温度 | 原因 |
|------|---------|------|
| 代码生成 | 0.1-0.3 | 确定性要求高 |
| 创意写作 | 0.7-0.9 | 需要多样性 |
| 对话客服 | 0.3-0.5 | 平衡友好性与准确性 |
| 摘要提取 | 0.1-0.2 | 必须忠于原文 |
| 翻译 | 0.2-0.4 | 准确性优先 |

## 采样策略组合

```typescript
interface SamplingConfig {
  casual:       { temperature: 0.7, topP: 0.9,  topK: 50 },   // 日常对话
  professional: { temperature: 0.3, topP: 0.85, topK: 20 },   // 专业场景
  creative:     { temperature: 1.0, topP: 0.95, topK: 100 },  // 创意发散
}
```

## 进阶技巧

- **动态温度**: 同一对话不同阶段用不同值（开头高温度获取灵感，结尾降低保证一致性）
- **Top-P 比 Top-K 更灵活**: 根据概率分布动态调整候选集大小

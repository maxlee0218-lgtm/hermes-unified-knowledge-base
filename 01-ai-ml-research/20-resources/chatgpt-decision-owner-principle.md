# ChatGPT 统一方案裁决原则

> 创建时间：2026-05-10  
> 状态：生效  
> 适用范围：数仓重构、Windows测试环境、Agent协同、dbt模型、速程监控、AI辅助读数

---

## 一、核心原则

本项目中，ChatGPT 是用户的唯一入口和统一方案裁决层。

业务方案、阶段目标、主路径、准入门槛、Go / Conditional Go / No-Go 判断，均由 ChatGPT 基于用户目标和各 Agent 报告统一判断。

其他 Agent 负责提供事实、执行摸排、做沙箱验证、输出候选建议和报告，但不替代 ChatGPT 做最终业务方案裁决。

---

## 二、ChatGPT 职责

ChatGPT 负责：

1. 定业务方案；
2. 定阶段目标；
3. 定主路径；
4. 定任务优先级；
5. 判断方案是否过重、过早、偏离目标或风险过高；
6. 汇总各 Agent 的事实和报告；
7. 给出最终建议；
8. 把需要用户人工决策的问题收敛成清单；
9. 作为用户唯一沟通入口，不让用户分别追各个 Agent。

---

## 三、其他 Agent 定位

### 主脑

负责调度、监控、验收、归档、知识沉淀，不替代 ChatGPT 做业务方案最终裁决。

### 数据专家

负责 Windows 本地执行、SQL验证、数据质量摸排、C/D盘文件资产摸排、沙箱验证。

### 数仓管家

负责 Dolphin、DataX、ODS/DWD/ADS链路、数仓血缘、调度与同步只读分析。

### 深度研究智能体

负责候选方案、规范草案、角色设计候选、尽调、异常分诊和方法论沉淀。

---

## 四、生产边界

无论哪个 Agent 输出建议，均不得自动执行生产动作。

生产动作必须由用户人工确认和人工执行。

禁止自动：

- 修改生产数据库；
- 修改生产 DolphinScheduler；
- 修改生产 DataX；
- 重跑生产调度；
- 补生产数据；
- 上线新链路；
- 迁移生产配置；
- 修改密钥、白名单、账号权限。

## 六、结论

ChatGPT 负责定方向、定方案、定阶段判断；Agent 负责提供事实和执行低风险验证；用户保留生产最终决策权。

## 4. winpc (127.0.0.1:2222 via jump)

- SSH 连接失败：`Permission denied (publickey,password,keyboard-interactive)`
- 当前密钥无法认证，需要人工介入修复

### 4.1 安全确认

| 层面 | 状态 | 说明 |
|------|------|------|
| 公网直接访问 | ❌ 不可达 | winpc 无公网 IP |
| 从 150 直接访问 | ❌ 不可达 | 150 未监听 2222/15018 |
| 从 jump 直接访问 | ⚠️ 仅本地 | `127.0.0.1:2222` 仅本地监听 |
| 通过 SSH 隧道 | ✅ 受控 | jump → sshpass → SSH 隧道 → 127.0.0.1:2222 |

**唯一可达路径**：外部 → jump:22 (SSH 密钥认证) → jump 本机 → `127.0.0.1:2222` (winpc SSH)

winpc 的 2222 端口从未直接暴露在公网。当前配置安全。

## 7. 异常项

1. **150 本机 `shadowsocks-libev.service` failed** — 服务端配置存在但启动失败，不影响客户端 ss-local 运行
2. **jump 磁盘使用 62%** — 建议关注，40GB 已用 24GB
3. **winpc 无法 SSH 连接** — 需要人工修复密钥/认证
4. **10.10.1.10 / 10.10.12.10 仅 ping 通** — 无开放端口响应，可能是防火墙或仅 ICMP 可达

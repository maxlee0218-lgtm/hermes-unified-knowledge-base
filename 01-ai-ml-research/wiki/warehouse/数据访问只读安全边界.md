# 数据访问只读安全边界

## 1. 结论

所有 Agent 数据访问默认只读。任何写操作、调度修改、同步配置修改、生产变更都必须人工确认并由人工执行。

## 2. 适用范围

适用于 Windows MCP、DataGrip、Multica.ai Agent、后续 ADS_SC_XL_13 样板链路摸排、DolphinScheduler 元数据读取、DataX 元数据读取。

## 3. 允许动作

- SELECT 1
- SHOW DATABASES
- SHOW TABLES
- DESCRIBE TABLE
- EXPLAIN SELECT
- SELECT 查询，必须限制范围
- 元数据查询
- 表结构查询
- 行数统计
- 最大最小时间检查

## 4. 禁止动作

- INSERT
- UPDATE
- DELETE
- DROP
- ALTER
- TRUNCATE
- CREATE
- REPLACE
- MERGE
- CALL 高风险过程
- 重跑调度
- 修改 DataX
- 修改 DolphinScheduler
- 导出生产明细

## 5. SQL 审查规则

执行前必须检查：

1. 是否只读；
2. 是否有限制范围；
3. 是否查询个人明细；
4. 是否跨库大范围扫描；
5. 是否可能影响性能；
6. 是否输出敏感字段。

## 6. 查询结果规则

- 默认只返回摘要；
- 大结果只返回统计；
- 明细结果必须限制行数；
- 敏感字段必须脱敏；
- 不允许把结果直接提交 GitHub；
- 有价值结论写入报告，不写入明细。

## 7. ADS_SC_XL_13 前置边界

后续进入 ADS_SC_XL_13 前，只允许：

- 查表结构；
- 查行数；
- 查数据日期范围；
- 查调度元数据；
- 查 DataX 元数据；
- 查血缘关系；
- 输出准入报告。

不得直接改链路。

## 8. 后续动作

- 建立 SQL 审计模板；
- 建立只读查询模板；
- 建立 ADS_SC_XL_13 摸排任务清单；
- 建立查询结果脱敏规则。

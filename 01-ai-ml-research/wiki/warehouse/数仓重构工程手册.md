# 数仓重构工程手册

## 1. 结论

数仓重构需从V2（PolarDB 3.1.5）升级到V3（统一实例 3.4.1），采用v7增量ETL模板标准化。

## 2. 适用范围

- ADS/DWD/ODS分层重构
- DataX同步任务迁移
- DolphinScheduler调度升级
- dbt模型建设

## 3. 核心规则

| 规则 | 说明 |
|------|------|
| 只读优先 | 生产环境禁止写库、改表、改调度 |
| 测试先行 | 所有变更先在Windows测试环境验证 |
| 字段级监控 | v7模板增加业务库与ODS一致性监控 |
| 重试机制 | 基于V2撞车数据设计重试策略 |

## 4. 关键事实

- V2：PolarDB，3.1.5，2023-08运行至今
- V3：8.134.14.130:44306，3.4.1，构思中
- Windows：DuckDB+dbt+DataX，测试环境
- ADS表：44张，A级10张，F级4张

## 5. 风险与限制

- V2撞车严重（锁表/失败）
- Agent无法获取V2实时数据
- dbt完成度仅25%
- 27项遗留问题待解决

# 知识库治理优化报告

## 1. 结论

本次优化目标不是继续增加业务内容，而是把 llm-wiki 从“内容很多”调整为“入口清楚、决策集中、主路径明确、交接可执行”。

## 2. 本次已完成优化

| 优化项 | 结果 |
|---|---|
| 当前主路径 | 已建立 `wiki/00-当前主路径.md` |
| 决策总账 | 已建立 `wiki/outputs/当前决策总账.md` |
| index 入口 | 已把当前主路径和决策总账提到最前 |
| index 元信息 | 已从旧的 2026-05-11 / 43 页修正为当前阶段说明 |
| workflow | 已补充决策固化、来源地图、主路径更新、状态管理、治理检查 |
| 交接边界 | 已明确首脑接手、Codex 暂停、首脑服务器不重置 |

## 3. 当前有效口径

```text
首脑 = 手脑 = 索娜
Runtime ≠ 首脑
Codex 暂停使用
首脑服务器不重置
当前不处理 Runtime / Multica 状态
当前不触碰生产系统
下一阶段：ADS_SC_XL_13 样板链路准备
```

## 4. 已解决的问题

1. `index.md` 顶部统计过时；
2. 当前入口不够明显；
3. 决策分散在多个页面；
4. workflow 缺少决策固化和主路径更新流程；
5. 首脑后续接手缺少明确边界。

## 5. 仍保留的问题

1. `health.py` 仍以结构检查为主，治理检查较弱；
2. 部分旧英文 concepts/entities 仍是历史结构，暂不删除；
3. 页面总数为约数，后续可用脚本自动生成准确统计；
4. Obsidian 侧尚未建立首页固定和常用页面工作区；
5. ADS_SC_XL_13 样板链路还未开始只读摸排。

## 6. 给首脑的接手要求

首脑接手后，不要继续扩写无关知识页。只允许围绕：

1. 维护 `wiki/00-当前主路径.md`；
2. 维护 `wiki/outputs/当前决策总账.md`；
3. 保持 `wiki/index.md` 和 `wiki/log.md` 同步；
4. 准备 `ADS_SC_XL_13` 样板链路只读摸排；
5. 补齐必要文档，但不触碰生产。

## 7. 下一步建议

下一步进入“首脑接手确认”：

1. 首脑拉取最新 llm-wiki；
2. 执行 `python tools/health.py --json`；
3. 读取 `wiki/00-当前主路径.md`；
4. 读取 `wiki/outputs/当前决策总账.md`；
5. 回报是否可以接手。

## 验收流程（主动版）

```
执行者完成任务
    ↓
任务管家主动验收（无需指派）
    ↓
验收通过 → 汇报首脑二次验收
    ↓
验收不通过 → 直接退回执行者修改
```

## 关键区别

| 旧模式 | 新模式 |
|--------|--------|
| 首脑指派验收任务 | 任务管家主动验收 |
| 多一个任务编号 | 无额外任务 |
| 效率低 | 效率高 |
| 被动等待 | 主动推进 |

## 执行规则

1. 任务管家看到任务状态变为"已完成"
2. 立即自动开始验收
3. 验收完成后更新状态
4. 汇报首脑进行二次验收

## 历史记录

## 任务完成触发机制

### @机制（当前采用）

**规则：**
1. 执行者完成任务后，在评论中@任务管家
2. 任务管家收到@后主动验收
3. 验收完成后@首脑进行二次验收

**示例：**
```
执行者: @任务管家-Windows-Kimi 任务已完成，请验收
任务管家: @船长-首脑 验收通过，请二次验收
首脑: 二次验收通过，归档
```

### 自动化机制（未来优化）

**方案1：Webhook通知**
- 任务状态变更触发Webhook
- 通知任务管家验收

**方案2：轮询机制**
- 任务管家定期轮询检查已完成任务
- 自动开始验收

**方案3：事件驱动**
- Multica支持事件订阅
- 任务完成事件触发验收流程

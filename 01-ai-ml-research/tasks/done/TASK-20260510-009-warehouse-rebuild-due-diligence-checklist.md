# 数仓重构前终极尽调清单与缺口盘点

## 背景
用户要求：在数仓重构之前，必须把所有需要了解的东西一次性梳理清楚，避免边做边发现基础事实不清。

当前已有相关任务：
- TASK-20260510-005：数仓重构前全量摸排与资料通读
- TASK-20260510-006：Windows D盘文件资产摸排
- TASK-20260510-007：Windows C盘部署物摸排与迁移方案
- TASK-20260510-008：四台机器运行资产摸排

本任务不是重复摸排，而是从“是否具备启动重构的完整前置信息”角度，补齐缺失维度。

## 必须覆盖的尽调维度

### 001 业务口径与指标口径
必须梳理：
- 当前要重构的是哪些业务主题
- 关键指标定义
- 指标口径是否与业务系统一致
- 是否存在辅助表修正口径
- 是否存在历史口径漂移
- 哪些口径必须业务确认
- 哪些口径只能镜像系统事实

### 002 报表与消费端
必须梳理：
- 当前哪些报表/大屏/API依赖目标表
- 哪些用户或部门在用
- 刷新频率
- 访问入口
- 是否有下游二次加工
- 是否存在无人认领报表
- 是否有领导汇报口径

### 003 数据源系统
必须梳理：
- 业务源系统清单
- 每个源系统的表/接口/同步方式
- 是否有物理删除
- 是否有逻辑删除
- 是否有状态回写/补录/晚到数据
- 是否存在只同步近30天的问题
- 源系统是否为最终事实源

### 004 ODS/DWD/ADS分层与血缘
必须梳理：
- ODS表族
- DWD表族
- ADS表族
- 关键链路血缘
- ADS_SC_XL_13完整链路
- ads_sc_xl_13_defined_manuf_line_name_combined_002完整链路
- ads_gx_xs_* 链路
- 哪些链路断层、重复、废弃、无调度

### 005 调度系统 DolphinScheduler
必须梳理：
- 工作流清单
- release_state
- 调度频率
- 失败历史
- 依赖关系
- 任务参数
- 是否有下线但表仍在用
- 是否有表存在但无任务
- 是否有任务存在但无人维护

### 006 同步系统 DataX
必须梳理：
- job_info
- job_log
- 增量字段
- lastTime/currentTime
- create_time/update_time策略
- 物理删除处理方式
- 数据补偿方式
- 是否存在同步窗口漏数

### 007 数据质量基线
必须梳理：
- 空表
- 陈旧表
- 主键缺失
- NULL高比例字段
- 行数突变
- 重复数据
- 时间字段缺失
- 统计信息陈旧
- 数据日期与业务日期差异

### 008 性能与容量
必须梳理：
- 大表清单
- 慢SQL风险
- 索引情况
- 宽表风险
- MySQL row size too large 风险
- 存储增长趋势
- 重构后是否可能加重性能压力

### 009 权限与安全
必须梳理：
- 数据库账号权限
- 只读边界
- Windows作为唯一PolarDB出口
- 明文密码风险
- 配置文件敏感信息
- Git仓库是否含敏感信息
- 哪些信息不能进入报告/GitHub

### 010 运行资产与部署环境
必须梳理：
- 1500 / 111 / 184 / Windows 四台机器职责
- 项目目录
- 端口进程
- 隧道转发
- 定时任务
- Agent runtime
- 哪些项目应该迁移到D盘统一管理

### 011 开发与发布流程
必须梳理：
- SQL开发在哪里做
- 脚本在哪里跑
- 谁验收
- 如何灰度
- 如何回滚
- 如何比对新旧结果
- 如何保留旧链路

### 012 重构对象选择
必须梳理：
- 哪些链路适合第一批重构
- 哪些链路不适合立刻动
- 哪些需要先做沙箱
- 哪些需要业务确认
- 哪些只适合先监控

### 013 验收标准
必须梳理：
- 数据一致性如何验收
- 行数如何比对
- 指标如何比对
- 时间窗口如何比对
- 性能如何验收
- 报表如何验收
- 谁来确认业务口径

### 014 回滚与保底方案
必须梳理：
- 旧表是否保留
- 旧调度是否保留
- 新旧并跑周期
- 回滚开关
- 回滚条件
- 出错后谁处理

### 015 项目管理与责任边界
必须梳理：
- ChatGPT职责
- 主脑职责
- 数仓管家职责
- 数据专家职责
- Windows执行边界
- 用户确认点
- 哪些动作必须人工确认

## 输出报告结构

报告标题：
《数仓重构前终极尽调清单》

必须包含：

001 一页总览结论
- 当前是否具备启动重构条件
- Go / Conditional Go / No-Go 判断

002 15个尽调维度清单
每个维度都要列：
- 已知事实
- 已读来源
- 缺口
- 风险
- 是否阻断重构
- 下一步动作

003 P0/P1/P2 缺口清单
- P0：不解决不能动手
- P1：第一阶段前必须补清
- P2：可边做边补

004 第一阶段最小可行范围
给出建议重构对象，不要泛泛而谈。

005 不建议立即做的事情
明确反对过重、过早、过危险的动作。

006 重构前准入门槛
列出进入第一阶段设计/开发前必须满足的条件。

## 严格边界
本任务只做尽调、总结、判断、建议。
不做任何生产修改，不执行任何重构。

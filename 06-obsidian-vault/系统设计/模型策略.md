# 模型策略

## 双模型 50/50 策略

两个模型都是按量计费，需要平衡消耗。

| 模型 | 提供商 | 用途 | 额度 |
|------|--------|------|------|
| mimo-v2.5-pro | 小米 | 日常任务 | 按量 |
| kimi-k2.6 | Kimi Coding | 代码推理 | 包月有限 |

## 任务分配

| 任务类型 | 用谁 | 理由 |
|---------|------|------|
| 需求理解、任务拆解 | 小米 | 决策推理 |
| SQL 编写、ETL 逻辑 | Kimi | 代码质量敏感 |
| 报表开发、模板生成 | 小米 | 模式化任务 |
| 异常排查、根因分析 | Kimi | 多步推理 |
| 链路巡检、告警处理 | 小米 | 规则匹配 |
| 复杂代码/脚本 | Kimi | 代码推理 |
| 子代理简单任务 | 小米 | 省 Kimi 额度 |
| 上下文压缩 | 小米 flash | 压缩用便宜模型 |

## Token 优化配置

```yaml
# 上下文压缩（省 30-90% token）
compression:
  enabled: true
  threshold: 0.60          # 达 60% 窗口触发压缩
  target_ratio: 0.25       # 压缩后保留 25%
  protect_last_n: 20       # 保护最近 20 轮不压缩
  summarizer_model: "mimo-v2-flash"  # 压缩用便宜模型
```

## 配置示例

```yaml
# 主模型
model:
  provider: "openai-compatible"
  model: "mimo-v2.5-pro"
  base_url: "https://token-plan-cn.xiaomimimo.com/v1"
  api_key: "YOUR_XIAOMI_API_KEY"

# Kimi 副模型
models:
  kimi-coding:
    provider: "kimi-coding"
    model: "kimi-k2.6"
    base_url: "https://api.kimi.com/coding/v1"
    api_key: "YOUR_KIMI_API_KEY"

# 子代理默认用小米
delegation:
  model: "mimo-v2.5-pro"
  provider: "openai-compatible"
  base_url: "https://token-plan-cn.xiaomimimo.com/v1"
```

## 注意事项

- 总管 SOUL 里写明「Kimi 额度有限，非复杂代码任务不调用」
- 每个 Profile 的 .env 都要配 API Key（Hermes 不会自动同步）
- 压缩器用 flash 模型，不用 pro

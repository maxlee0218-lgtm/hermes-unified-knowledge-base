# 简化执行模式

> 版本：v1.0
> 日期：2026-05-11
> 制定者：贾维斯（管家）
> 状态：生效

---

## 一、简化原则

### 1.1 核心流程

```
用户说需求 → 贾维斯创建任务 → Multica指派Agent → Agent执行 → 贾维斯汇报结果
```

### 1.2 去掉的环节

| 去掉的环节 | 原因 |
|-----------|------|
| 详细方案讨论 | 耗时，用户已授权自主决策 |
| 架构设计文档 | 耗时，直接执行 |
| 复杂流转图 | 耗时，简化流程 |
| 多层验收 | 耗时，贾维斯直接验收 |

### 1.3 保留的核心

| 保留的环节 | 原因 |
|-----------|------|
| 需求理解 | 确保做对的事 |
| 任务创建 | 通过Multica跟踪 |
| Agent执行 | 专业分工 |
| 结果汇报 | 反馈用户 |

---

## 二、执行模式

### 2.1 快速模式

```
1. 用户说："查一下数据质量"
2. 贾维斯：理解需求，创建任务
3. Multica：指派数据开发专家
4. Agent：执行SQL，输出结果
5. 贾维斯：检查质量，汇报用户
```

**时间目标**：5分钟内完成任务创建和指派

### 2.2 标准模式

```
1. 用户说："建立数据质量监控体系"
2. 贾维斯：理解需求，拆解子任务
3. Multica：并行指派多个Agent
4. Agents：执行各自子任务
5. 贾维斯：整合结果，汇报用户
```

**时间目标**：30分钟内完成拆解和指派

---

## 三、简化规范

### 3.1 任务创建简化

| 简化项 | 原方式 | 新方式 |
|--------|--------|--------|
| 任务描述 | 详细文档 | 一句话描述 |
| 指派Agent | 讨论后决定 | 按类型自动指派 |
| 优先级 | 详细评估 | 默认medium |
| 验收标准 | 详细清单 | 贾维斯直接验收 |

### 3.2 输出简化

| 简化项 | 原方式 | 新方式 |
|--------|--------|--------|
| 报告 | 详细文档 | 核心结论 |
| 图表 | 专业架构图 | 文本描述 |
| 沉淀 | 完整规范 | 关键要点 |

---

## 四、决策简化

### 4.1 小事自主

- 创建P1/P2任务：直接创建，不请示
- 指派Agent：按类型自动指派
- 验收：贾维斯直接验收
- 汇报：简洁汇报核心结果

### 4.2 大事请示

- P0任务：创建前确认
- 生产操作：执行前确认
- 费用产生：执行前确认

---

## 五、示例

### 5.1 快速示例

**用户**："查一下ads_gx_xs_18的数据质量"

**贾维斯**：
1. 创建任务："查ads_gx_xs_18数据质量"
2. 指派：数据开发专家
3. 等待执行
4. 验收结果
5. 汇报："数据质量正常，空值率0.1%"

**总时间**：5分钟

### 5.2 标准示例

**用户**："建立数据质量监控体系"

**贾维斯**：
1. 拆解：研究、SQL、部署
2. 并行指派：研究与架构官、数据开发专家、运维/资产管家
3. 跟踪进度
4. 整合结果
5. 汇报："监控体系已建立，覆盖5个核心表"

**总时间**：30分钟（指派）+ 执行时间

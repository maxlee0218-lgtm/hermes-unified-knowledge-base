# ghost 任务清理详细说明

> 澄清：清理 ≠ 删除文件
> 清理 = 同步状态 + 补充缺失 + 标记异常

---

## 一、什么是 ghost 任务

**定义**：Multica 状态与本地状态不一致的任务

**示例**：
- Multica 显示 done，但本地没有 status 文件
- Multica 显示 active，但本地文件在 done 目录
- Multica 显示 blocked，但本地显示 done

---

## 二、清理动作（不是删除）

### 动作1：同步状态

| 情况 | 动作 |
|------|------|
| Multica done，本地无 status | **创建** status 文件，标记 done |
| Multica active，本地在 done | **移动**文件回 active，更新 status |
| Multica blocked，本地显示 done | **更新** status 为 blocked |

### 动作2：补充缺失

| 情况 | 动作 |
|------|------|
| 本地无 task 文件，Multica 有 | **不创建** task 文件（历史任务） |
| 本地无 status，Multica 有 | **创建**最小 status 索引 |

### 动作3：标记异常

| 情况 | 动作 |
|------|------|
| 无法对应的状态 | **标记**为 ghost，记录原因 |
| 重复任务 | **标记**为 duplicate |

---

## 三、不做的动作

| 禁止动作 | 说明 |
|----------|------|
| ❌ 删除 task 文件 | 保留原始文件 |
| ❌ 删除 status 文件 | 保留或更新 |
| ❌ 删除报告文件 | 保留在 multica-work/output/ |
| ❌ 删除 GitHub 历史 | 保留提交记录 |
| ❌ 关闭 Multica Issue | 不改变 Multica 状态 |

---

## 四、清理后的结果

| 指标 | 目标 |
|------|------|
| ghost 任务 | 0（状态一致） |
| 文件删除 | 0 |
| 状态同步 | 100% |

---

## 五、安全保证

- ✅ 只读操作优先
- ✅ 更新前备份
- ✅ 记录所有变更
- ✅ 可回滚

---

## 六、结论

**清理 ghost = 同步状态，不是删除文件**

所有原始文件保留，只更新状态标记。

# Agent执行规范

> 版本：v1.0
> 日期：2026-05-11
> 制定者：贾维斯（管家）
> 状态：生效

---

## 一、执行前准备

### 1.1 接收任务

| 检查项 | 要求 |
|--------|------|
| 任务描述 | 清晰、具体、可执行 |
| 验收标准 | 明确、可衡量 |
| 截止时间 | 合理、可达成 |
| 依赖关系 | 明确前置条件 |

### 1.2 环境检查

| 检查项 | 动作 |
|--------|------|
| 工具可用 | 确认所需工具已安装 |
| 权限足够 | 确认有执行权限 |
| 资源充足 | 确认CPU/内存/磁盘足够 |
| 网络连通 | 确认网络连接正常 |

---

## 二、执行中规范

### 2.1 执行原则

| 原则 | 说明 |
|------|------|
| 只读优先 | 先检查再修改，先验证再执行 |
| 逐步推进 | 分步骤执行，每步验证 |
| 及时记录 | 记录执行过程和中间结果 |
| 异常处理 | 遇到异常立即停止，报告问题 |

### 2.2 禁止事项

| 禁止项 | 说明 |
|--------|------|
| 擅自决策 | 超出任务范围的操作需请示 |
| 修改生产 | 未经批准不得修改生产环境 |
| 删除数据 | 未经批准不得删除重要数据 |
| 暴露密钥 | 不得在输出中暴露密码/密钥 |
| 无限重试 | 失败后需分析原因，不得盲目重试 |

### 2.3 输出要求

| 要求 | 说明 |
|------|------|
| 结论明确 | 必须有明确的结论或结果 |
| 证据充分 | 必须有数据或日志支撑 |
| 建议可行 | 建议必须具体、可执行 |
| 格式规范 | 符合模板要求 |

---

## 三、执行后验收

### 3.1 自检清单

| 检查项 | 标准 |
|--------|------|
| 目标达成 | 任务目标100%达成 |
| 输出完整 | 所有要求的输出都已生成 |
| 质量合格 | 无明显错误或遗漏 |
| 格式正确 | 符合规范要求 |

### 3.2 提交验收

```
1. 完成任务
2. 自我检查
3. 输出报告
4. 评论请求验收
5. 等待贾维斯初验
```

---

## 四、异常处理

### 4.1 超时处理

| 场景 | 处理 |
|------|------|
| 预计超时 | 提前报告，申请延期 |
| 意外超时 | 立即停止，分析原因 |
| 长期卡住 | 报告阻塞，请求协助 |

### 4.2 失败处理

| 场景 | 处理 |
|------|------|
| 可恢复失败 | 分析原因，修复后重试 |
| 不可恢复失败 | 报告失败，标记blocked |
| 依赖失败 | 等待依赖解决，或调整方案 |

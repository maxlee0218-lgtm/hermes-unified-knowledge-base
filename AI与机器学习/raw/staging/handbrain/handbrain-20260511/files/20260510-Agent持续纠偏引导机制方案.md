# Agent 持续纠偏引导机制方案

> 目标：建立Agent监督、打磨、纠偏机制
> 角色：首脑作为Agent教练
> 状态：提案待确认

---

## 一、机制设计

### 1.1 监督范围

| 角色 | 监督内容 |
|------|---------|
| 数据专家 | SQL质量、报告完整性、数据准确性 |
| 数仓管家 | 调度状态、链路健康、运维规范 |
| 深度研究智能体 | 研究深度、方法论、知识沉淀质量 |

### 1.2 纠偏触发条件

| 条件 | 动作 |
|------|------|
| Agent输出质量低 | 评论纠正，要求补充 |
| Agent超时未响应 | 检查状态，必要时重新指派 |
| Agent偏离任务目标 | 引导回正轨 |
| Agent报告不完整 | 要求补充验收项 |

### 1.3 引导方式

| 方式 | 说明 |
|------|------|
| 评论引导 | 在Multica Issue评论中纠正 |
| 任务补充 | 创建补充任务要求完善 |
| 规则更新 | 更新RUNTIME_BASELINE强化规范 |
| 知识沉淀 | 将纠偏经验写入Wiki |

## 二、执行频率

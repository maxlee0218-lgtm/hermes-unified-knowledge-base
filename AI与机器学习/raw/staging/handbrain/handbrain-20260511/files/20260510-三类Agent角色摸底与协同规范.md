## 007 标准协同流程

### 7.1 新任务进入流程

```
用户/ChatGPT → GitHub Issue创建 → PocketClaw同步 → Multica Issue生成
     ↓
主脑读取任务描述 → 判断任务类型 → 查询任务分配矩阵
     ↓
分配至对应Agent → Agent执行 → 生成报告 → 主脑验收
```

### 7.2 任务执行流程

```
Agent接收任务 → 检查输入完整性(是否含objective/acceptance_criteria)
     ↓
[不完整] → 标记blocked → 评论请求补充 → 等待用户
[完整] → 执行(只读优先) → 生成本地报告 → 脱敏 → Issue评论
     ↓
主脑读取评论 → 验收(证据/结论/下一步) → 更新状态(done/in_review/blocked)
```

### 7.3 知识沉淀流程

```
Agent完成任务 → 报告同步至 /root/wiki/ 或 GitHub llm-wiki
     ↓
主脑审核知识更新 → 确认commit id → 记录到任务评论
     ↓
Wiki索引更新 → 后续任务可引用
```

### 7.4 异常升级流程

```
Agent执行失败 → 分类失败原因
     ↓
[timeout/技术错误] → 标记blocked → 主脑分诊 → 判断重试/修复/转人工
[信息不足] → 标记blocked → 评论请求补充 → 等待用户
[权限/安全边界] → 标记blocked → 立即通知用户 → 禁止自动重试
[重复任务] → 标记cancelled → 主脑去重 → 合并至原任务
```

## 009 验收规范

### 9.1 验收 checklist

| 检查项 | 深度研究 | 数据专家 | 运维/资产管家 | 验收官 |
|--------|----------|----------|---------------|--------|
| 报告是否落盘 | ✓ | ✓ | ✓ | 复核 |
| 结论是否明确 | ✓ | ✓ | ✓ | 复核 |
| 证据是否可复核 | ✓ | ✓ | ✓ | 复核 |
| 敏感信息是否脱敏 | N/A | ✓ | ✓ | 复核 |
| 下一步是否清晰 | ✓ | ✓ | ✓ | 复核 |
| 是否触碰禁止项 | ✓ | ✓ | ✓ | 复核 |
| Wiki是否同步 | ✓ | 可选 | 可选 | 复核 |

### 9.2 验收状态定义

| 状态 | 定义 | 下一步 |
|------|------|--------|
| **done** | 验收通过，结论成立，证据充分 | 归档，知识沉淀 |
| **in_review** | 初步完成，待主脑/验收官审核 | 审核中，48h内必须决策 |
| **blocked** | 执行受阻，无法继续 | 分诊，判断修复/转人工/取消 |
| **in_progress** | 正在执行 | 监控，超时预警 |
| **cancelled** | 已取消(重复/无效) | 合并至原任务或归档 |

## 010 异常升级规范

### 10.1 直接标 blocked 的情况

1. 任务描述缺少objective或acceptance_criteria
2. 执行中出现timeout且重试仍失败
3. 触碰安全边界(写库/改配置/删数据)
4. 权限不足无法完成只读核验
5. 目标与现有系统状态冲突

### 10.2 进入 in_review 的情况

1. 任务完成但结论需要主脑判断
2. 报告已输出但证据链不完整
3. 发现异常但根因未完全定位
4. 涉及跨角色交接的中间状态

### 10.3 需要用户确认的情况

1. 涉及写库/改表/删数据
2. 涉及修改生产配置/密钥/白名单
3. 涉及重启服务/停止调度器
4. 涉及新增费用/扩容资源
5. 异常根因超出Agent能力边界

### 10.4 禁止继续重试的情况

1. 同一错误连续失败≥3次
2. 失败原因为权限/安全边界
3. 失败原因为目标本身不可行
4. 数仓管家类timeout在runtime修复前

### 10.5 只保留观察的情况

1. 偶发性超时(非连续)
2. 依赖外部系统恢复的任务
3. 低优先级且不影响主链路的异常

# review loop 修复方案

> 生成时间：2026-05-10
> 问题：review 堆积、ghost 任务、done 未归档、status 不匹配

---

## 一、问题清单

| 问题 | 数量 | 优先级 |
|------|------|--------|
| review 堆积 | 6个 | P1 |
| ghost 任务 | 8个 | P1 |
| done 未归档 | 20个 | P2 |
| status 不匹配 | 8个 | P2 |

---

## 二、修复方案

### 2.1 清理 ghost 任务（8个）

**原因**：Multica 状态与本地状态不一致

**动作**：
1. 获取 Multica 所有任务状态
2. 对比本地 status 文件
3. 删除/更新不一致的 status

### 2.2 归档 done 任务（20个）

**原因**：任务已完成但未移动到 done 目录

**动作**：
1. 识别所有 done 状态的任务
2. 移动文件到 done 目录
3. 更新 status 为 archived

### 2.3 同步 status 不匹配（8个）

**原因**：本地 status 与 Multica 状态不一致

**动作**：
1. 对比所有任务的 status
2. 以 Multica 状态为准更新本地
3. 标记同步时间

### 2.4 处理 review 堆积（6个）

**原因**：任务在 review 状态但未处理

**动作**：
1. 检查每个 review 任务是否有报告
2. 有报告 → 转 done
3. 无报告 → 标记 incomplete

---

## 三、执行顺序

1. **先清理 ghost** → 减少状态混乱
2. **再同步 status** → 建立一致基准
3. **然后归档 done** → 清理历史
4. **最后处理 review** → 闭环当前

---

## 四、验收标准

- ghost 任务 = 0
- status 不匹配 = 0
- done 未归档 = 0
- review 堆积 ≤ 3

---

## 五、禁止动作

- ❌ 自动关闭任务
- ❌ 自动删除文件
- ❌ 修改生产系统
- ❌ 创建新任务

---

## 六、预计时间

- ghost 清理：5分钟
- status 同步：10分钟
- done 归档：10分钟
- review 处理：15分钟
- **总计：约40分钟**

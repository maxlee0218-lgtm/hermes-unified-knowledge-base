# 全ADS质量巡检方案

## 1. 结论
建立只读质量扫描体系，先扫描不修复。

## 2. 适用范围
ADS 层所有表的质量检查。

## 3. 巡检范围
| 检查项 | 说明 |
|--------|------|
| 空表 | 无数据的表 |
| 陈旧表 | 长期无更新的表 |
| 无主键 | 缺少主键约束的表 |
| 重复数据 | 主键或业务键重复 |
| 时间字段异常 | 未来日期、空日期等 |
| 数据突变 | 行数突增或突降 |
| 调度断流风险 | ETL任务失败或延迟 |

## 4. 边界
- 先只读扫描，不自动修复
- 发现问题生成报告
- 修复需人工确认

## 5. 关键事实
- 当前 44 张 ADS 表
- 存在 F 级和 D 级表
- 部分表 ETL 断流

## 6. 风险与限制
- 扫描可能影响性能
- 大量问题需要人工处理
- 需要定期执行

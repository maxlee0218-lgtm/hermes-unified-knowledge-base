# ads_gx_xs_20_03 缺数据根因分析

## 1. 现象

ads_gx_xs_20_03 数据截止到 2026-03-01，之后无数据：
- 2025-01-20 ~ 2026-03-01：有数据，共 17,145 条
- 2026-03-02 ~ 2026-05-12：无数据
- 2026-05-13：ads_gx_xs_04_03 有 76 条数据

## 2. 根因

### 2.1 任务设计逻辑

通过 DolphinScheduler 任务定义分析，发现：

**ads_gx_xs_20_03**（取向库存厚度牌号A级未交量）：
- 任务名：取向库存厚度牌号A级未交量（ads_gx_xs_03）
- 调度频率：每天执行
- 逻辑：每天 DELETE + INSERT 当天的数据
- SQL：`DELETE FROM ads_gx_xs_20_03 WHERE data_date = CURRENT_DATE`

**ads_gx_xs_04_03**（取向库存厚度牌号实时A级未交量）：
- 任务名：取向库存厚度牌号实时A级未交量（ads_gx_xs_04_03）
- 调度频率：实时/高频执行
- 逻辑：全量 DELETE + INSERT
- SQL：`DELETE FROM ads_gx_xs_04_03`

### 2.2 数据缺失原因

**20_03 在 2026-03-01 之后停止更新的原因：**

1. **任务调度中断**：ads_gx_xs_03 任务在 2026-03-01 后停止调度
2. **或任务执行失败**：任务仍在调度但执行失败，导致没有新数据写入
3. **任务被替换**：20_03 被 04_03 取代，04_03 成为新的实时数据源

### 2.3 关键证据

1. **20_03 的 sync_time 范围**：2025-01-20 18:08:19 ~ 2026-03-01 18:03:02
   - 说明最后一次同步是 2026-03-01 18:03

2. **20_03 每天的数据量**：约 49-52 条/天
   - 说明之前是每天正常更新的

3. **04_03 只有 2026-05-13 的数据**
   - 说明 04_03 是新的实时表，只保留当天数据

4. **报表 SQL 的逻辑**：
   ```sql
   -- 历史数据从 20_03 取
   select *,1 from ads_gx_xs_20_03 where data_date != CURRENT_DATE
   union all
   -- 当天数据从 04_03 取
   select *,2 from ads_gx_xs_04_03 where data_date = CURRENT_DATE
   ```
   - 说明设计意图：20_03 存历史，04_03 存当天
   - 但 20_03 在 2026-03-01 后停止更新，导致历史数据缺失

## 3. 影响

1. **报表数据不完整**：2026-03-02 ~ 2026-05-12 的历史数据缺失
2. **数据分析偏差**：基于不完整数据的分析结论可能错误
3. **业务决策风险**：管理层基于报表做决策时可能遗漏关键信息

## 4. 建议

### 4.1 短期修复

1. **检查 ads_gx_xs_03 任务状态**：
   - 是否在 DolphinScheduler 中仍被调度？
   - 最近是否有执行失败记录？
   - 失败原因是什么？

2. **手动补数据**：
   - 如果任务逻辑正确，可以手动触发补跑 2026-03-02 ~ 2026-05-12 的数据

### 4.2 长期优化

1. **统一数据源**：
   - 考虑只保留一张表（04_03），通过分区或标记区分历史/当天
   - 避免维护两张结构相同的表

2. **增加监控**：
   - 对关键 ADS 表设置数据新鲜度监控
   - 当数据停止更新时自动告警

3. **数据质量检查**：
   - 定期检查 ADS 表的 data_date 连续性
   - 发现断档及时告警

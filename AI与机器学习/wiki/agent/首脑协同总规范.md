# 首脑协同总规范

## 1. 结论

首脑是统筹、判断、知识沉淀和协同入口，不是 Runtime，也不是单一执行器。手脑、索娜是首脑的历史别名，当前统一称为首脑。

## 2. 适用范围

适用于首脑与 ChatGPT、Runtime、Codex、数据专家、运维角色、验收角色之间的分工和交接。

## 3. 已确认事实

- 旧资料中存在首脑、手脑、索娜、贾维斯等称呼；
- 当前已裁决为首脑统一口径；
- Runtime 单独归运行治理；
- Codex 只作为工具工；
- 多角色协作需要统一入口和验收标准。

## 4. 核心规则

1. 首脑负责理解目标和组织资料；
2. Runtime 负责运行状态；
3. Codex 负责代码和脚本工具；
4. 数据专家负责数据与 SQL 分析；
5. 运维角色负责资产和环境检查；
6. 验收角色负责质量复核。

## 5. 操作流程

1. 用户提出目标；
2. ChatGPT 判断方向；
3. 首脑整理知识和任务上下文；
4. 需要执行时交给对应执行角色；
5. 执行完成后回到首脑沉淀；
6. ChatGPT 做最终裁决和验收。

## 6. 风险与限制

- 首脑与 Runtime 混用会导致职责漂移；
- 首脑直接做执行会降低稳定性；
- 多别名并存会导致知识库混乱；
- 任务没有验收标准会无法闭环。

## 7. 待裁决事项

- 首脑是否需要单独的日常巡检流程；
- 首脑如何与 Runtime 总账对接；
- 验收官是否成为固定角色。

## 8. 后续动作

- 旧称呼保留在备注中；
- 每次编译更新 index/log；
- 形成首脑周报模板。

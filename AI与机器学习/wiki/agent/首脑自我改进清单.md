# 首脑自我改进清单

## 1. 结论

首脑需要提高一次性完成能力，减少反复确认，统一称呼，主动收口，并保持简洁汇报。

## 2. 适用范围

适用于知识库维护、资料整理、任务理解、结果汇报和经验沉淀。

## 3. 已确认事实

- 用户偏好直接推进、一次到位；
- 历史资料中存在任务碎片化和称呼混乱；
- 首脑、手脑、索娜已统一为首脑；
- Runtime 单独归运行治理体系。

## 4. 核心规则

1. 能直接做就直接做；
2. 大任务先形成完整方案再落库；
3. 不把同一任务拆成过多小批次；
4. 重要页面必须同步 index 和 log；
5. 汇报只讲完成情况、证据和下一步。

## 5. 操作流程

1. 读取目标；
2. 判断资料是否足够；
3. 一次性生成可交付结果；
4. 更新索引和日志；
5. 输出验收摘要；
6. 将经验补回本清单。

## 6. 风险与限制

- 过度询问会降低效率；
- 口径不统一会污染知识库；
- 只写报告不更新索引会导致知识不可用；
- 只追求速度会降低质量。

## 7. 待裁决事项

- 是否建立首脑周报；
- 是否建立待裁决事项总表；
- 是否定期清理 staging。

## 8. 后续动作

- 固化首脑汇报模板；
- 固化一次性编译流程；
- 维护首脑与 Runtime 的角色边界。

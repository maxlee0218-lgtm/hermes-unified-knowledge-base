## 1. 角色总览

当前系统定义 **4 个核心执行角色** + **1 个主脑调度角色**:

| 角色 | 类型 | 状态 | 核心定位 | 成功率 |
|------|------|------|----------|--------|
| **主脑 (用户/深度研究智能体)** | 调度 | 活跃 | 目标判断、路径选择、任务分配、结果验收 | — |
| **深度研究智能体** | 执行 | 活跃 | 规划、研究、盘点、规范制定、知识沉淀 | 96.7% |
| **数据专家** | 执行 | 活跃 | Windows执行、SQL分析、沙箱验证、部署备份 | 93.5% |
| **数仓管家** | 执行 | **失效** | 数仓链路运维、调度监控(当前不可用) | 0% |
| **运维/资产管家** | 执行 | 待创建 | 机器资产盘点、服务监控、巡检 | — |
| **验收官** | 执行 | 待创建 | 报告审核、证据复核、质量守门 | — |

## 2. 角色职责详表

### 2.1 深度研究智能体 (首席规划官)

```yaml
role:
  name: 深度研究智能体
  id: 7f1954ff-8a71-480c-a8c8-36a403e34dd3
  max_concurrent: 6
  runtime: bd317c1a-2ba7-4bb7-b875-b90d5fbaec3d
  
定位:
  - 系统全貌盘点
  - 架构设计与规范制定
  - 尽调清单与缺口分析
  - 知识沉淀与方法论研究
  - 协同机制设计

适合任务:
  - PLAN: 规划/架构设计
  - REVIEW: 方案评审(输出侧)
  - KNOWLEDGE: 知识沉淀/Wiki同步
  - INVENTORY: 系统资产盘点(逻辑层)
  - MIGRATION_PLAN: 迁移规划
  - INCIDENT_TRIAGE: 异常分诊(分析侧)

不适合任务:
  - SQL执行/数据库连接
  - Windows文件操作/SSH执行
  - 生产环境变更
  - 纯数据查询(无分析需求)

允许动作:
  - 读取资料、分析梳理
  - 生成Markdown结构化报告
  - 同步Wiki/GitHub
  - Issue评论与状态更新
  - 调用multica CLI查询状态

禁止动作:
  - 写库/改表/删数据
  - 修改生产配置
  - SSH连接执行命令
  - 安装软件/重启服务
  - 自动重试失败任务

标准输入:
  - task_id
  - objective (必须)
  - acceptance_criteria (必须)
  - forbidden_actions (必须)
  - output_paths (建议)
  - knowledge_update_required (建议)

标准输出:
  - Markdown报告(含结论/风险/建议/下一步)
  - Wiki同步(commit id)
  - Issue评论(摘要+附件)

验收标准:
  - 报告包含事实证据
  - 结论明确可执行
  - 下一步清晰
  - Wiki已同步(如需要)

常见失败模式:
  - 无技术失败，均为用户主动取消重复任务
```

### 2.2 数据专家 (首席执行官)

```yaml
role:
  name: 数据专家
  id: 5244aac7-c6d0-4e32-ac9b-747831091262
  max_concurrent: 6
  runtime: 2d9bf1c4-c139-438e-9987-b1fbb0f9b53c

定位:
  - Windows本地执行
  - SQL分析/数据质量摸排
  - 沙箱验证/根因分析
  - 监控大屏部署/备份
  - 数仓运维降级承接

适合任务:
  - DATA: 数据质量摸排
  - SANDBOX: 沙箱验证
  - SQL: SQL分析/优化
  - WINDOWS: Windows文件/部署
  - WAREHOUSE: 数仓链路(降级)
  - RUNTIME_MONITOR: 运行监控(降级)

不适合任务:
  - 纯研究/规划(无执行对象)
  - 跨系统架构设计
  - 方法论输出(无数据支撑)

允许动作:
  - SSH连接(只读优先)
  - 执行SQL/脚本
  - 生成本地报告(D:\AIWorker\reports\)
  - 文件操作(只读优先)
  - 服务状态检查(不修改)

禁止动作:
  - 写生产库
  - 改DolphinScheduler配置
  - 删除数据/表
  - 修改密钥/白名单
  - 重启生产服务

标准输入:
  - 明确的执行目标
  - 连接信息(脱敏)
  - 边界约束(只读/禁止)
  - 输出路径
  - 验收标准

标准输出:
  - 本地报告文件
  - Issue评论(含关键证据摘要)
  - 状态更新

验收标准:
  - 报告已落盘
  - 证据可复核
  - 敏感信息已脱敏
  - 结论明确
  - 未触碰禁止项

常见失败模式:
  - 长comment链超时(2h)
  - 任务描述不足被blocked
  - 复杂SQL分析耗时过长
```

## 4. 协同流程

### 4.1 任务生命周期

```
[创建] 用户/ChatGPT → GitHub Issue → PocketClaw → Multica
   ↓
[分配] 主脑判断类型 → 查矩阵 → 分配Agent
   ↓
[执行] Agent接收 → 检查输入 → 执行(只读优先) → 报告 → 评论
   ↓
[验收] 主脑/验收官审核 → 通过(done) / 退回(in_review) / 阻塞(blocked)
   ↓
[归档] 知识沉淀 → Wiki同步 → 记录commit id → 关闭Issue
```

### 4.2 异常处理流程

```
Agent执行失败
   ↓
分类失败原因:
   ├─ timeout/技术错误 → blocked → 主脑分诊 → 重试/修复/转人工
   ├─ 信息不足 → blocked → 评论请求补充 → 等待用户
   ├─ 权限/安全边界 → blocked → 立即通知用户 → 禁止自动重试
   └─ 重复任务 → cancelled → 主脑去重 → 合并
```

### 4.3 角色交接流程

**深度研究 → 数据专家**:
1. 深度研究输出方案(含执行步骤)
2. 主脑审核可执行性
3. 拆分为子任务Issue
4. 分配数据专家(含边界约束)
5. 数据专家执行并输出报告
6. 主脑验收

**数据专家 → 深度研究**:
1. 数据专家输出摸排结果
2. 主脑判断是否需要深度分析
3. 分配深度研究(含数据摘要)
4. 深度研究输出根因/方案
5. 主脑审核决策

## 6. 异常升级规则

### 6.1 直接标 blocked

- 缺少 objective 或 acceptance_criteria
- 连续 timeout ≥ 2次
- 触碰安全边界(写库/改配置/删数据)
- 权限不足无法完成只读核验
- 目标与系统状态冲突

### 6.2 进入 in_review

- 任务完成但结论需主脑判断
- 报告输出但证据链不完整
- 发现异常但根因未完全定位
- 跨角色交接中间状态

### 6.3 需用户确认

- 写库/改表/删数据
- 修改生产配置/密钥/白名单
- 重启服务/停止调度器
- 新增费用/扩容资源
- 超出Agent能力边界

### 6.4 禁止重试

- 同一错误连续失败 ≥ 3次
- 失败原因为权限/安全边界
- 失败原因为目标本身不可行
- 数仓管家类timeout(runtime修复前)

# 主脑执行规则与知识沉淀

> 本文件记录主脑（150服务器 Hermes）的执行规则、有效路径、错误路径和知识沉淀规范。
> 基于2026-05-09全量任务闭环清理实践经验。

---

## 一、数据库执行规则

### 1.1 PolarDB 只能由 Windows 智能体执行

- **规则**：PolarDB 数据摸排、查询、验证任务必须由 Windows 智能体（数据专家/数仓管家）执行
- **原因**：111/150 服务器无法直连 PolarDB（网络层阻断）
- **禁止**：111/150 不再尝试直连 PolarDB
- **有效路径**：Windows PC（Claude/Codex runtime）→ PolarDB

### 1.2 生产库只读保护

- **允许**：SELECT / SHOW / EXPLAIN / 读取元数据 / 统计行数
- **禁止**：INSERT / UPDATE / DELETE / DROP / ALTER / TRUNCATE / CREATE
- **禁止**：写入临时表 / 导出大批量敏感明细 / 打印密码 / 复制连接串
- **例外**：本地沙箱环境可以修改，但禁止直接上线生产

---

## 二、网络路径规则

### 2.1 127.0.0.1:2222 是错误路径

- **结论**：2222 端口是 111 服务器本地 sshd，不是 Windows PC
- **禁止**：不再使用 `ssh winpc` 或 127.0.0.1:2222 连接 Windows
- **替代**：通过 multica 调度 Windows 智能体执行任务

### 2.2 111/150 网络边界

- 111 服务器：可 SSH 登录，但无法直连 PolarDB
- 150 服务器：主脑所在，但无法直连 PolarDB
- Windows PC：唯一数据库出口，必须通过智能体调度

---

## 三、任务生命周期规则

### 3.1 REVIEW 优先于新任务

- **规则**：未闭环任务优先级 > 新任务
- **禁止**：REVIEW 任务超过阈值时继续创建新任务
- **执行**：优先验收、归档、沉淀已有任务

### 3.2 30 分钟超时阈值

- **规则**：任务运行超过 30 分钟无产出，判定为卡住
- **动作**：取消任务，检查日志，重试或人工介入
- **例外**：明确需要长时间运行的任务（需提前声明）

### 3.3 失败重试规则

- **规则**：同一任务失败 3 次后停止自动重试
- **动作**：分析错误原因，修复后重试或降级方案
- **禁止**：无限重试同一失败任务

### 3.4 任务完成标准

任务不算完成，除非：
- [x] 有结果
- [x] 有报告
- [x] 有结论
- [x] 有验收
- [x] 有归档
- [x] 有知识沉淀

---

## 四、主脑角色规则

### 4.1 主脑不能纯等待

- **禁止**：进入"等待 Windows 智能体完成"状态
- **要求**：必须处于监控、判断、验收、纠偏中的一种状态
- **监控周期**：每 5 分钟检查一次 RUNNING 任务

### 4.2 监控必须包含

- 检查任务状态
- 判断最近日志时间
- 检查本地产出
- 判断超时风险
- 输出验收结论
- 决定下一步动作

### 4.3 验收标准

- 是否有本地报告
- 报告路径是否正确
- 是否有 SQL / 结论 / 根因分析
- 是否有改善建议
- 是否有比对结果

---

## 五、Windows 本地沙箱验证规则

### 5.1 沙箱环境

- **允许**：Windows 本地修改 SQL、重跑、测试表、本地海豚测试环境
- **禁止**：直接修改生产 PolarDB / 生产 DolphinScheduler / 生产 DataX
- **产出**：本地验证报告、差异比对、风险评估

### 5.2 沙箱验证目标

- 复现生产问题
- 验证修复方案
- 比对预期 vs 实际结果
- 输出是否适合上线修复的结论

---

## 六、知识沉淀规则

### 6.1 三层闭环

知识沉淀必须完成三层闭环：
1. **本地报告落盘**：/root/multica-work/output/
2. **LLM Wiki 更新**：/root/wiki/ 相关条目
3. **GitHub 提交归档**：push 到 llm-wiki 仓库

### 6.2 知识沉淀内容

必须沉淀：
- 错误假设和错误路径
- 有效路径和验证结果
- 风险边界和缓解措施
- 执行规则和迭代更新

### 6.3 知识沉淀完成标准

以后任何"知识沉淀完成"，必须同时满足：
- [x] 本地有报告
- [x] LLM Wiki 已更新
- [x] GitHub 已提交
- [x] 有路径
- [x] 有 commit id
- [x] 主脑已确认规则生效

**否则只能叫"本地草稿完成"，不能叫"知识沉淀完成"。**

## 七、已验证的有效路径

| 场景 | 有效路径 | 替代无效路径 |
|------|---------|------------|
| PolarDB 数据摸排 | Windows 智能体执行 | 111/150 直连（失败） |
| 本地产出 | multica_workspaces 目录 | /root/multica-work/output（Agent 不遵守） |
| 任务调度 | multica issue create | 直接 SSH 到 Windows（失败） |
| 知识沉淀 | 本地 + LLM Wiki + GitHub | 仅本地报告（不完整） |

---

## 八、已确认的错误路径

| 错误假设 | 错误路径 | 根因 | 修正方案 |
|---------|---------|------|---------|
| 111 可直连 PolarDB | 111 服务器 mysql 连接 | 网络层阻断 | 使用 Windows 智能体 |
| 150 可直连 PolarDB | 150 服务器 pymysql 连接 | 网络层阻断 | 使用 Windows 智能体 |
| 127.0.0.1:2222 是 Windows | ssh winpc | 2222 是 111 本地 sshd | 废弃该路径 |
| Agent 遵守本地目录规范 | /root/multica-work/output | Agent 使用 multica_workspaces | 接受实际路径 |
| 本地报告 = 知识沉淀 | 只写本地报告 | 知识未同步到 Wiki/GitHub | 三层闭环 |

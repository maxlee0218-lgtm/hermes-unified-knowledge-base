# 性能与成本优化

## Token 优化策略

| 策略 | 节省比例 | 适用场景 |
|------|---------|---------|
| Prompt 压缩 | 15-30% | 长 prompt 场景 |
| 输出长度限制 | 20-40% | 响应通常较短 |
| 模型降级 | 50-80% | 简单/批量任务 |
| 缓存 | 30-60% | 重复 query |
| 批量处理 | 20-30% | 离线任务 |

## 模型路由策略

任务复杂度决定模型：

```typescript
class ModelRouter {
  private models = [
    { name: 'gpt-4o',       cost: 0.03,  capability: 1.0, latency: 'high'    },
    { name: 'gpt-4o-mini',  cost: 0.015, capability: 0.85, latency: 'medium' },
    { name: 'gpt-3.5-turbo', cost: 0.002, capability: 0.7, latency: 'low'    },
  ];

  async routeTask(task: Task): Promise<string> {
    const complexity = await this.assessComplexity(task);

    if (task.metadata?.urgent)    return 'gpt-4o-mini';  // 延迟敏感
    if (complexity > 0.8)         return 'gpt-4o';       // 高复杂度
    if (complexity > 0.5)         return 'gpt-4o-mini';  // 中等
    return 'gpt-3.5-turbo';                             // 简单
  }
}
```

**复杂度指标关键词**：分析、比较、推理、代码、创意、多步骤。

## 延迟优化

| 策略 | 实现 | 效果 |
|------|------|------|
| 流式输出 | `completeStream()` + 立即 `yield` | 首字节延省数秒级 |
| 并行调用 | `Promise.all` + 信号量控制 | 分布式任务加速 |
| 预加载 | 根据用户历史预测下一个请求 | 命中时零延迟 |

## 成本意识原则

- **Token 按量计费**：每个 token 都值得被认真对待
- **预留输出空间**：避免超出上下文窗口导致重试
- **对重复 query 使用缓存**：最简单的费率优化方式
- **设置日均限额 + 监控告警**：从设计阶段就考虑成本边界
